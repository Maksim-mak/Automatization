1 баг-репорт:
Заголовок:
 Метод `capitalize` некорректно обрабатывает строки с ведущими пробелами.

Описание
 Метод `capitalize` класса `StringUtils` не удаляет ведущие пробелы перед капитализацией первой буквы строки. Это приводит к некорректному результату, когда пробел сохраняется в начале и буква после пробела не становится заглавной.

Шаги для воспроизведения
1. Создайте экземпляр класса `StringUtils`.
2. Вызовите метод `capitalize` со строкой, содержащей ведущий пробел:
   ```python
   utils = StringUtils()
   result = utils.capitalize(" python")
   ```
3. Сравните результат с ожидаемым значением `" Python"`.

Ожидаемый результат
Метод `capitalize` должен вернуть строку `" Python"` (с заглавной буквой "P" и пробелом перед ней).

Фактический результат
Метод `capitalize` возвращает строку `" python"` (с символом пробела и маленькой буквой "p").

Примечания
- Этот дефект возникает только при передаче строки, содержащей ведущие пробелы.
- Для строк без ведущих пробелов метод работает корректно.




2 баг-репорт:

Заголовок
Метод `delete_symbol` некорректно обрабатывает строки, состоящие только из пробелов.

Описание
 Метод `delete_symbol` в классе `StringUtils` удаляет все пробелы из строки, состоящей только из пробелов, когда символ, который нужно удалить, также является пробелом.

Шаги для воспроизведения
1. Создайте экземпляр класса `StringUtils`.
2. Вызовите метод `delete_symbol` со строкой, состоящей исключительно из пробелов:
   ```python
   utils = StringUtils()
   result = utils.delete_symbol("   ", " ")
   ```
3. Сравните результат с ожидаемым значением `"   "`.

Ожидаемый результат
Метод `delete_symbol` должен вернуть строку, состоящую из пробелов: `"   "`.

Фактический результат
Метод `delete_symbol` возвращает пустую строку: `""`.

Примечания
- Этот дефект возникает только при передаче строки, состоящей исключительно из пробелов.
- Может потребоваться дополнительное определение поведения в зависимости от требований к функционалу.
